我是機器人 隱私權政策 (Privacy Policy)

最後更新日期：2025年10月12日

1. 收集的資訊類型 (Information We Collect)

**我是機器人** 只收集服務運作所必需的非個人識別資訊（NPII）。我們不會儲存您的電子郵件、IP 地址或密碼。

1.1 機器人功能資料 (Bot Functionality Data)

為了提供核心功能，我們可能會儲存以下資訊：

  • **使用者 ID (User ID)**：用於識別使用者、管理遊戲狀態（如 RPS 遊戲）、檢查管理員權限、以及作為指令紀錄的來源。
  • **伺服器 ID (Guild ID)**：用於區分不同的伺服器、儲存伺服器的設定（如通知頻道 ID、歡迎訊息等）以及管理伺服器的指令紀錄。
  • **頻道 ID (Channel ID)**：用於儲存伺服器設定中指定的通知頻道、歡迎頻道或反應身分組的頻道。
  • **訊息 ID (Message ID)**：用於追蹤反應身分組的訊息，以及在 `/delete` 等指令中處理訊息。
  • **使用者指令內容 (Command Content)**：在執行指令時，如 `/say` 的內容、`/calc` 的表達式、`/抽籤` 的選項，會被臨時處理或儲存於 **指令紀錄 (`command_logs`)** 中。

1.2 網頁後台資料 (Web Dashboard Data)

當您使用我們的網頁後台時，我們會透過 Discord OAuth2 流程存取以下資料：

  • **您的 Discord 使用者資訊 (User Data)**：包括您的使用者名稱、頭像和 ID。這些資料僅用於顯示您的儀表板、驗證您是否為管理員，並且不會被永久儲存於我們的資料庫中（僅儲存於 Flask Session）。
  • **您擁有的伺服器清單 (Guilds Data)**：包括伺服器的名稱、ID 和您在該伺服器的權限等級。這是為了讓您可以在儀表板上選擇要管理的伺服器。

2. 資料用途 (How We Use the Information)

我們收集的資訊僅用於以下目的：

  • **提供和維護服務：** 使 Discord 機器人指令（如踢出、封鎖、語音播放）和網頁設定功能正常運作。
  • **伺服器配置：** 儲存並套用您在網頁後台中設定的伺服器專屬配置。
  • **問題診斷與安全：** 臨時性的指令紀錄 (`command_logs`) 用於追蹤服務錯誤、解決技術問題和防止濫用。

3. 資料儲存與保留 (Data Storage and Retention)

  • **伺服器配置資料** (Guild ID, Channel ID, 設定訊息)：將永久儲存於您的資料庫中，直到您將機器人從伺服器中移除，或手動要求刪除。
  • **指令紀錄** (`command_logs`)：僅在應用程式運行時臨時儲存於伺服器記憶體中，並限制最多 **100 條紀錄**。應用程式重啟後將被清除。
  • **網頁後台 Session 資料** (OAuth 數據)：僅在您瀏覽網頁後台時儲存於 Flask Session 中，您登出或 Session 到期後即自動刪除。

4. 資料刪除權利 (Your Right to Deletion)

您可以隨時要求我們刪除所有與您的 Discord 伺服器或使用者 ID 相關的永久儲存資料。請通過下方的聯繫方式提出刪除請求。在收到請求後的 **7 天** 內，我們將從我們的資料庫中刪除您的資料。

5. 條款變更 (Changes to this Policy)

我們保留隨時更新本隱私權政策的權利。更新後的版本將在本頁面上發布。
